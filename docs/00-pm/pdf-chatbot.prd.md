## Executive Summary

| 관점 | 내용 |
|------|------|
| **Problem** | 근로기준법 등 노무 관련 법률 PDF는 방대하고 어려워, 담당자가 조항을 직접 찾는 데 많은 시간을 낭비함 |
| **Solution** | PDF를 서버에서 읽어 AI 컨텍스트로 제공하고, 문서 내용에만 근거한 정확한 답변을 대화 형태로 제공 |

| **Core Value** | "내 PDF"를 AI 전문가로 만들어 법 조항 검색 시간을 90% 단축하고, 할루시네이션 없는 신뢰 가능한 답변 제공 |

## 1. 기회 발견 (Opportunity Discovery)

### 1.1 목표 성과 (Desired Outcome)

노무 담당자가 법 조항 확인에 소요하는 시간을 주당 평균 2시간 이상 단축하고,  
잘못된 정보로 인한 노무 리스크를 줄인다.

### 1.2 핵심 아이디어 (Top 5)

| # | 아이디어 | 관점 | 근거 |
|---|----------|------|------|
| 1 | PDF 내용만 근거로 답변 (RAG 없이 전문 삽입) | 신뢰성 | 단일 PDF, 단순 구조에서 가장 정확 |
| 2 | 누구나 본인 PDF로 교체 가능한 구조 | 확장성 | `docs/` 폴더만 바꾸면 다른 법령에도 적용 |
| 3 | API 키 서버 전용 처리 | 보안 | Vercel 환경 변수로 키 보호 |
| 4 | 말풍선 형태 채팅 UI | UX | 친숙한 메신저 인터페이스로 진입 장벽 낮춤 |
| 5 | 스트리밍 응답으로 기다리는 체감 시간 단축 | 성능 | 첫 글자부터 즉시 표시 |

### 1.3 기회 솔루션 트리 (Opportunity Solution Tree)

```
목표: 노무 법령 조항 확인 시간 단축 + 정보 신뢰도 향상
│
├── 기회 1: 조항을 찾는 데 너무 오래 걸림
│   ├── 해결책 A: PDF 전문을 AI 컨텍스트로 제공 → 질문하면 바로 답변 ✅ (선택)
│   └── 해결책 B: 조항 번호로 직접 검색하는 검색 엔진 구축
│
├── 기회 2: AI 답변을 신뢰하기 어려움 (할루시네이션)
│   ├── 해결책 C: 시스템 프롬프트에 "PDF 내용에만 근거" 지시 삽입 ✅ (선택)
│   └── 해결책 D: 출처 조항 번호를 함께 표시
│
└── 기회 3: 회사마다 다른 내부 규정이 있음
    └── 해결책 E: docs/ 폴더 교체로 다른 PDF 적용 가능한 구조 ✅ (선택)
```

## 2. 가치 제안 및 전략 (Value Proposition & Strategy)

### 2.1 JTBD 가치 제안 (6-Part)

| 파트 | 내용 |
|------|------|
| **Who (누구)** | 노무 법령을 자주 확인해야 하는 HR 담당자, 소규모 사업주 |
| **Why (왜)** | 근로기준법 조항을 찾아 해석하는 데 시간과 전문 지식이 필요함 |
| **What Before (이전 상황)** | PDF 직접 열람, 구글 검색, 노무사 문의 — 느리고 비용이 발생 |
| **How (방법)** | PDF를 AI에 통째로 제공하고, 질문에 해당 조항을 인용하여 즉시 답변 |
| **What After (이후 상태)** | 30초 내에 정확한 조항 기반 답변, 노무사 문의 건수 감소 |
| **Alternatives (대안 대비)** | 일반 ChatGPT는 할루시네이션 위험, 노무사 앱은 유료·느림 |

**가치 제안 한 문장**: "내 PDF를 AI 전문가로 만들어, 법 조항을 초 단위로 확인하고 할루시네이션 없는 신뢰 가능한 답변을 받는다."

### 2.2 린 캔버스 (Lean Canvas)

| 섹션 | 내용 |
|------|------|
| **Problem** | ① PDF 직접 열람은 느림, ② AI 답변은 신뢰하기 어려움, ③ 노무사 문의는 비용이 큼 |
| **Solution** | ① PDF 전문을 AI 컨텍스트로 제공, ② "PDF 내용만 근거" 지시, ③ 교체 가능한 PDF 구조 |
| **UVP** | 할루시네이션 없이 내 PDF에서만 답변하는 노무 AI 챗봇 |
| **Unfair Advantage** | PDF 교체만으로 다른 회사·법령에 즉시 적용 가능한 오픈소스 구조 |
| **Customer Segments** | 50인 미만 사업장 HR 담당자, 스타트업 CEO, 총무/인사팀 |
| **Channels** | GitHub 오픈소스 공유, 노무사 커뮤니티, 스타트업 슬랙 채널 |
| **Revenue Streams** | 오픈소스 (무료), 향후 SaaS 전환 가능 |
| **Cost Structure** | Claude API 호출 비용 (gpt-4o-mini — 저렴), Vercel 호스팅 (무료 티어) |
| **Key Metrics** | 일일 질문 수, 응답 만족도, PDF 교체 횟수 |

### 2.3 SWOT 분석

| | 유리 | 불리 |
|---|------|------|
| **내부** | **강점**: 단순 구조, PDF 교체 용이, Claude haiku-4-5 저비용·고품질 | **약점**: 단일 PDF만 지원, 대화 기록이 서버에 저장 안 됨 |
| **외부** | **기회**: 노동법 개정 시 PDF만 교체, 다른 법령 챗봇으로 확장 | **위협**: OpenAI/Claude 직접 사용 증가, ChatGPT PDF 플러그인 |

**SO 전략**: Claude haiku-4-5의 저비용 + PDF 교체 용이성으로 다양한 법령 버전 빠르게 출시  
**WT 전략**: 오픈소스로 공개해 직접 경쟁 대신 생태계 기여

### 3.2 경쟁 환경

| 경쟁자 | 강점 | 약점 | 우리의 기회 |
|--------|------|------|-------------|
| ChatGPT (PDF 업로드) | 유명도, UI 완성도 | 할루시네이션 위험, 유료(Plus), 내부 배포 어려움 | 오픈소스로 자체 배포 가능, 비용 없음 |
| Claude.ai (PDF) | 정확도 높음, 한글 우수 | 유료, API 직접 사용 불가 | API로 커스텀 배포 + 코드 공개 |

**차별화 전략**: "내 PDF + 오픈소스 + 할루시네이션 방지" 조합. 자체 서버에 배포 가능하여 기업 내부 도구로 활용 가능.

## 5. 제품 요구사항 (PRD)

### 5.1 요약

PDF 기반 노무 FAQ 챗봇은 `docs/` 폴더의 PDF를 서버 시작 시 자동으로 읽어  
Claude gpt-4o-mini API를 통해 말풍선 형태 채팅 UI에서 스트리밍 답변을 제공한다.  
API 키는 Vercel 서버리스 함수 내에서만 처리하며, 프론트엔드에 절대 노출되지 않는다.

### 5.2 배경 및 맥락

- **왜 지금인가**: Claude gpt-4o-mini 모델이 200K 토큰 컨텍스트 윈도우 지원 → 긴 PDF 전문 삽입 가능
- **무엇이 바뀌었나**: LLM API 비용 급감 → 소규모 팀도 자체 AI 도구 구축 가능
- **왜 Claude인가**: 한국어 품질 우수, gpt-4o-mini는 저비용·빠른 응답, Anthropic의 Constitutional AI로 안전성 높음

### 5.3 목표 및 핵심 결과 (OKR)

| 목표 | 핵심 결과 | 목표값 |
|------|----------|--------|
| 빠른 법 조항 검색 | 첫 스트리밍 글자 표시 시간 | 3초 이내 |
| 할루시네이션 방지 | PDF 내용 근거 답변 비율 | 100% (시스템 프롬프트 강제) |
| 손쉬운 커스터마이즈 | PDF 교체 후 재시작 시간 | 1분 이내 |
| 보안 | API 키 프론트엔드 노출 여부 | 0건 |

### 5.4 대상 세그먼트

- **Primary**: 노무 법령을 빈번히 확인하는 HR 담당자 및 사업주
- **Secondary**: 자사 내규 PDF를 AI 챗봇으로 만들고 싶은 개발자

### 5.5 가치 제안 (Value Propositions)

- **Jobs**: 법 조항 빠른 검색, 복잡한 규정을 쉽게 이해
- **Gains**: 노무사 문의 없이 즉시 답변, 24시간 접근 가능, 무료
- **Pains Solved**: 할루시네이션 제거, PDF 직접 열람 불필요, API 키 노출 위험 없음

### 5.6 핵심 기능 (Key Features)

| 기능 | 설명 | 우선순위 | 해결하는 기회 |
|------|------|:--------:|--------------|
| PDF 자동 로드 | 서버 시작 시 `docs/` 폴더 PDF를 읽어 텍스트 변환 후 메모리 저장 | Must | 조항 검색 속도 |
| 할루시네이션 방지 | 시스템 프롬프트에 "PDF 내용에만 근거" 지시 + "모르면 모른다고 답변" | Must | AI 신뢰성 |
| PDF 교체 가능 구조 | `docs/` 폴더에 PDF만 교체하면 다른 법령에 즉시 적용 | Must | 확장성 |
| Vercel 서버리스 API | `POST /api/chat` — Vercel 함수로 동작, API 키 서버 전용 처리 | Must | 보안 |
